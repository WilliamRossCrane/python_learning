## About This Repo

This repository is my Python learning and teaching catalogue.

It is designed to show progress from small beginner programs to more structured backend projects.

As a future Digital Solutions teacher, this repo is also a way to model how students can build skills over time:

* start small
* practise basic syntax
* build confidence
* move into frameworks
* organise projects clearly
* use Git commits to show progress

This repo is not about one giant project.

It is about showing a clear learning journey.

---

## Learning Pathway

| Area           | What It Includes                                                 | Why Use It?                                                   |
| -------------- | ---------------------------------------------------------------- | ------------------------------------------------------------- |
| `FUNDAMENTALS` | Basic Python files using `main.py`                               | Learn core Python syntax before using frameworks              |
| `FLASK`        | Web apps, routes, templates, forms, CRUD, databases              | Learn how Python can build websites and small full-stack apps |
| `FASTAPI`      | REST APIs, JSON, validation, request bodies, API docs            | Learn how modern apps send and receive data                   |
| `DJANGO`       | Coming later: larger web apps, users, admin, structured projects | Learn bigger Python web development with more built-in tools  |

| Framework | Best For                               | Why You Would Use It                                                   |
| --------- | -------------------------------------- | ---------------------------------------------------------------------- |

## 1. Fundamentals

The `FUNDAMENTALS` folder is the starting point.

It teaches basic Python using simple `main.py` files.

No frameworks.
No servers.
No extra packages.

Just Python.

### What It Teaches

* `print()`
* comments
* variables
* strings
* numbers
* input
* if statements
* lists
* loops
* dictionaries
* functions
* debugging

### Why Start Here?

Before using Flask, FastAPI, or Django, students need to understand basic Python.

Fundamentals teaches how code works in the simplest way possible.

### Best For

* complete beginners
* Years 7–10 coding practice
* syntax basics
* small terminal programs
* building confidence

---

## 2. Flask

The `FLASK` folder teaches web apps with Python.

Flask is a lightweight web framework.

### What It Teaches

* routes
* web pages
* templates
* forms
* CRUD apps
* databases
* simple APIs
* backend basics

### Why Use Flask?

Flask is a good first web framework because it is simple and flexible.

It helps students see how Python can respond to web requests and display pages in a browser.

### Pros

* beginner-friendly
* simple to start
* flexible
* great for small projects
* good for learning how websites work

### Cons

* less structure than Django
* bigger apps need more planning
* you choose many tools yourself

### Best For

Learning how Python can build websites, forms, dashboards, and small full-stack apps.

---

## 3. FastAPI

The `FASTAPI` folder teaches APIs with Python.

FastAPI is a modern framework for building backend APIs.

### What It Teaches

* REST APIs
* JSON responses
* API routes
* path parameters
* query parameters
* request bodies
* validation
* automatic API docs

### Why Use FastAPI?

FastAPI is useful because many modern apps use APIs to send and receive data.

Web apps, mobile apps, dashboards, and games often talk to APIs behind the scenes.

### Pros

* modern
* fast
* clean syntax
* automatic docs
* great for APIs
* good for backend learning

### Cons

* not focused on HTML pages
* can feel different after Flask
* works best once you understand basic APIs

### Best For

Learning how modern backend APIs work.

---

## 4. Django

The `DJANGO` folder will come later.

Django is a larger Python web framework.

### What It Teaches

* larger web apps
* users and login systems
* databases
* admin panels
* structured projects
* full-stack web development

### Why Use Django?

Django gives you more built-in tools than Flask.

It is useful when building bigger web apps that need structure from the start.

### Pros

* lots built in
* strong structure
* good for larger projects
* built-in admin panel
* useful for database-heavy apps

### Cons

* more to learn at the start
* heavier than Flask
* can feel too big for small projects

### Best For

Learning larger, more structured Python web apps.

---

## Repo Purpose

This repo is both a learning space and a teaching catalogue.

It can be used to show students that coding skills build over time.

Small beginner projects matter because they become the foundation for bigger apps later.
